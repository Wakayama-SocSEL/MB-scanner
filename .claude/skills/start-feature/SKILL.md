# 実装開始（初期化・計画）ワークフロー

あなたはプロジェクトのリードエンジニアです。以下のルールに従い、Phase 1（初期化と計画）を遂行してください。

## 参照ドキュメント
- [実装ワークフロー詳細](references/workflow.md)

## 💡 開発の基本ルール
1. **合意形成の徹底**:
   - ユーザーの意図とのズレや手戻りを防ぐため、`plan.md` を作成してユーザーの明示的な許可を得るまでは、プロダクトコード（`mb_scanner/`）を一切変更しないでください。
2. **コンテキストの保持**:
   - 後続のフェーズや別のスキルがこれまでの文脈を正確に把握できるよう、ユーザーからの指示や変更内容は必ずタスクディレクトリ内の `prompt.md` に記録し続けてください。

## 開始手順 (Phase 1: Initialization & Planning)

ユーザーの要望を受け取ったら、以下のステップで初期化を行ってください。

1. **機能名の決定**:
   - ユーザーの要望内容から、適切で簡潔な英語の機能名（kebab-case）を生成してください。
   - 例: "CSVエクスポート" → `add-csv-export`、"バグ修正: 検索遅延" → `fix-search-latency`

2. **タスクディレクトリの作成**:
   - 決定した機能名を引数にして、以下のスクリプトを実行してください。
   ```bash
   uv run python .claude/skills/start-feature/scripts/init_task.py <GENERATED_KEBAB_NAME>
※スクリプトの実行に失敗した場合は、Bashツールを使って自力でディレクトリ構造を構築してください。
プランニングと承認:
生成された plan.md を読み込み、ユーザーの要望に合わせて具体的な実装ステップを追記・提案してください。
ユーザーに計画を提示し、実装に進んでよいか明示的な承認を求めてください。
Phase 2（実装）への引き継ぎ:
ユーザーから計画の承認を得たら、本スキルの役割は完了です。
実装を担当する implement-feature スキルを呼び出してバトンタッチしてください。
