# 実装プロセス (Standard Implementation Flow)

## Phase 1: 準備と計画 (Setup & Planning)
1. **初期化**:
   - `scripts/init_task.py` を実行し、`.ai/tasks/XXXX_機能名/` を作成する。
2. **要件記録**:
   - 生成された `prompt.md` にユーザーの要望を詳細に記録する。
3. **プランニング**:
   - 生成された `plan.md` に以下の内容を記述する：
     - 目的と背景
     - 変更するファイル一覧
     - 新規作成するクラス/関数
     - データベース変更の有無（ある場合は `database` スキルを参照）
4. **承認依頼**:
   - 「以下のプランで実装を進めてよろしいですか？」とユーザーに `plan.md` の内容を提示する。
   - **注意**: ここで許可が出るまで Phase 2 に進んではいけません。修正要望があれば `plan.md` を更新して再提示します。

## Phase 2: 実装 (Implementation)
承認が得られたら、以下のサイクルで実装を行います。

1. **コーディング**:
   - `coding-standards` スキルに従いコードを記述する。
   - 必要に応じて `prompt.md` に作業ログを追記する。
2. **テスト実装**:
   - `quality-check` スキルのガイドラインに従い、`tests/` にテストを追加する。
   - 外部通信が発生する箇所は必ずモック化する。

## Phase 3: 品質保証 (Quality Assurance)
実装の区切りごとに、以下のコマンドを実行して品質を担保してください。

1. **テスト実行**:
   ```bash
   uv run pytest
   ```
2. 自動修正とフォーマット:
3. 型チェック:
エラーが発生した場合は、修正 -> 再チェックのループを回して解消してください。

## Phase 4: 完了 (Completion)
1. **記録**: すべてのチェックがパスしたら、`prompt.md` に最終的な対応内容を記録する。
2. **完了報告と次のステップ**:
   - ユーザーに実装完了を報告する。
   - **重要**: ドキュメントへの反映を行うため、以下のコマンド順で実行するようユーザーに促してセッションを終了する。
     ```text
     実装が完了しました。コンテキストをリセットしてドキュメントを更新するために、以下を実行してください：
     1. /clear
     2. /reflect-changes
     ```
