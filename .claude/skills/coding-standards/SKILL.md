# コーディング規約と実装ガイド

新しいデータ構造の定義や、ファイル入出力処理を実装する際は、以下のガイドラインに従ってください。

## 参照ドキュメント
- [詳細ガイドライン](references/guidelines.md)

## 実装時の判断フロー

### 1. データモデルの選択
変数を定義する際は、以下の基準で型を選択してください。詳細は参照ドキュメントを確認すること。
- **外部データ/設定/バリデーションが必要**: `Pydantic` を使用
- **内部データ/軽量な戻り値**: `TypedDict` を使用
- **禁止**: `Any` 型の使用

### 2. ファイル形式の選択
データを保存・出力する際は、用途に応じて JSON か CSV を選択してください。
- **構造化データ/システム連携**: JSON
- **分析用/フラットデータ**: CSV

## 品質チェック
コード変更後は必ず以下のコマンドで型整合性を確認してください。
```bash
just typecheck
```
